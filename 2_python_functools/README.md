# Python: `functools`

Материалы к воркшопу про модуль стандартной библиотеки `functools`.

`functools` — это набор инструментов для работы с функциями: декораторы без потери метаданных, кэширование результатов, частичное применение аргументов, свёртка коллекций и простая диспетчеризация по типам.

## Что разбираем

На воркшопе разбираются основные инструменты `functools`:

- `wraps` — как писать декораторы, которые не ломают имя, документацию и отладку функции;
- `cache` и `lru_cache` — как ускорять повторные вычисления с помощью кэширования;
- `partial` — как заранее подставлять часть аргументов и получать новую функцию;
- `reduce` — как сворачивать последовательность значений в один результат;
- `singledispatch` — как выбирать реализацию функции по типу аргумента.

## Основная идея

Функции в Python — это объекты.

Их можно:

- передавать в другие функции;
- возвращать из функций;
- оборачивать декораторами;
- настраивать заранее;
- кэшировать;
- использовать как часть более сложной логики.

`functools` даёт готовые инструменты для таких задач.

## Практика

Во время воркшопа можно попробовать несколько заданий.

### 1. Починить декоратор

Есть декоратор, который работает, но ломает метаданные функции.

Нужно добавить `functools.wraps`, чтобы сохранились:

- `__name__`;
- `__doc__`;
- поведение `help()`;
- нормальная отладка.

### 2. Ускорить функцию через кэш

Нужно взять функцию с повторяющимися вычислениями и добавить:

```python
from functools import cache
```

или:

```python
from functools import lru_cache
```

После этого можно сравнить время выполнения до и после.

### 3. Сделать функцию через `partial`

Нужно взять обычную функцию и заранее зафиксировать часть аргументов.

Например:

```python
from functools import partial
import json

dump_pretty = partial(json.dumps, ensure_ascii=False, indent=2)
```

Теперь `dump_pretty()` — это уже настроенная версия `json.dumps()`.

### 4. Разобрать `reduce`

`reduce` можно использовать для свёртки коллекции в одно значение.

Например:

```python
from functools import reduce

numbers = [1, 2, 3, 4]

result = reduce(lambda acc, x: acc * x, numbers, 1)

print(result)
# 24
```

Но важно помнить: часто `sum`, `max`, `min`, `join` или обычный цикл читаются лучше.

### 5. Посмотреть `singledispatch`

`singledispatch` позволяет описать одну функцию с разным поведением для разных типов.

```python
from functools import singledispatch

@singledispatch
def normalize(value):
    return str(value)

@normalize.register
def _(value: str):
    return value.strip().lower()
```

```python
from functools import (
    wraps,
    cache,
    lru_cache,

)
```

## На что обратить внимание

- `wraps` почти всегда нужен при написании своих декораторов.
- Декоратор должен возвращать результат обёрнутой функции.
- Кэшировать стоит только функции, которые при одинаковых аргументах возвращают одинаковый результат.
- Аргументы кэшируемой функции должны быть hashable.
- `lru_cache(maxsize=...)` помогает ограничить рост кэша.
- `partial` полезен, когда заранее заданные аргументы делают код понятнее.
- `reduce` стоит использовать осторожно: не всегда он читабельнее обычного цикла.
- `singledispatch` удобен для простых случаев выбора поведения по типу.

## Когда это полезно

`functools` часто пригождается, когда нужно:

- написать свой декоратор;
- сохранить имя и документацию функции после обёртки;
- ускорить повторяющиеся вычисления;
- убрать дублирующиеся аргументы из вызовов;
- сделать более выразительный callback;
- аккуратно обработать разные типы входных данных.
