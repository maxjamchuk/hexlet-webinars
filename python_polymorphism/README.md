# Python: Полиморфизм

Материалы к вебинару по полиморфизму в Python.

Вебинар построен вокруг небольших учебных Python-примеров. На них удобно разобрать классический полиморфизм, общий интерфейс, утиную типизацию, проблему большого `if/elif`, рефакторинг к полиморфизму и практическое добавление нового поведения через новый класс.

## Что понадобится на вебинаре

Для участия в вебинаре достаточно файлов из этой папки и установленного Python.

| Файл | Для чего нужен |
|------|----------------|
| `01_classic_shapes.py` | Классический пример полиморфизма на фигурах. |
| `02_bad_task_renderer.py` | Пример функции, которая разрастается из-за `if/elif`. |
| `03_polymorphic_task_renderer.py` | Основной пример рефакторинга к полиморфизму. |
| `06_practice_add_json_renderer.py` | Практическое задание для студентов. |
| `06_practice_add_json_renderer_solution.py` | Готовое решение практического задания. |

Остальные файлы нужны для дополнительных объяснений и резерва, если останется время.

## Список файлов

| Файл | Описание |
|------|----------|
| `01_classic_shapes.py` | Классический пример: `Rectangle` и `Circle` имеют общий метод `area()`, а функция работает с ними одинаково. |
| `02_bad_task_renderer.py` | Плохой пример: функция `render_task()` сама знает все форматы вывода и выбирает поведение через `if/elif`. |
| `03_polymorphic_task_renderer.py` | Хороший пример: разные рендереры имеют общий метод `render()`, а `render_task()` не проверяет конкретный тип. |
| `04_duck_typing.py` | Короткий пример утиной типизации: Python пытается вызвать нужный метод во время выполнения. |
| `05_when_if_is_ok.py` | Примеры ситуаций, где `if` нормален и не требует полиморфизма. |
| `06_practice_add_json_renderer.py` | Файл-задание: нужно добавить `JsonRenderer`, не меняя функцию `render_task()`. |
| `06_practice_add_json_renderer_solution.py` | Решение задания с `JsonRenderer`. |
| `07_reserve_task_filters.py` | Резервный пример: фильтрация задач через разные фильтры с общим методом `match()`. |
| `README.md` | Описание материалов и порядок запуска. |
