# Python: Полиморфизм — учебные примеры для воркшопа Hexlet

## Цель

Показать полиморфизм как способ писать общий код для разных объектов через одинаковый интерфейс. Вместо проверки типа объекта — вызов одного и того же метода.

## Порядок запуска

Запускайте файлы по порядку:

```bash
python 01_classic_shapes.py
python 02_bad_task_renderer.py
python 03_polymorphic_task_renderer.py
python 04_duck_typing.py
python 05_when_if_is_ok.py
python 06_practice_add_json_renderer.py
python 06_practice_add_json_renderer_solution.py
```

## Главное про полиморфизм

- Разные объекты могут иметь одинаковый метод с разным поведением.
- Вызывающий код не проверяет тип объекта — он просто вызывает метод.
- Добавить новый вариант поведения = добавить новый класс, а не менять существующую функцию.
- В Python для этого не нужны абстрактные классы — достаточно чтобы у объекта был нужный метод (утиная типизация).
