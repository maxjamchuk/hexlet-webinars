# Python: `functools`

Материалы к вебинару про модуль стандартной библиотеки `functools`.

`functools` помогает писать более выразительный код вокруг функций: сохранять метаданные декораторов, кэшировать дорогие вычисления, заранее фиксировать часть аргументов и аккуратно работать с функциями как с объектами.

## Что разбираем

На вебинаре разбираются основные инструменты `functools`:

- `wraps` — как писать декораторы, которые не ломают имя, документацию и сигнатуру функции;
- `cache` и `lru_cache` — как кэшировать результат функции и ускорять повторные вычисления;
- `partial` — как заранее зафиксировать часть аргументов функции;
- `reduce` — как свернуть последовательность значений в один результат;
- `singledispatch` — как выбирать реализацию функции по типу аргумента.

## Основная идея

Функции в Python — это объекты. Их можно:

- передавать в другие функции;
- возвращать из функций;
- оборачивать декораторами;
- настраивать заранее;
- кэшировать;
- использовать как строительные блоки для более выразительного кода.

`functools` даёт готовые инструменты для этих задач.

## Полезные команды

Импорт основных инструментов:

```python
from functools import wraps, cache, lru_cache, partial, reduce, singledispatch
```

Проверка, что `wraps` сохранил метаданные функции:

```python
print(func.__name__)
print(func.__doc__)
```

Просмотр статистики кэша:

```python
func.cache_info()
```

Очистка кэша:

```python
func.cache_clear()
```

## На что обратить внимание

- `wraps` почти всегда нужен при написании своих декораторов.
- Кэшировать стоит только функции, где одинаковые аргументы дают одинаковый результат.
- Аргументы кэшируемой функции должны быть hashable.
- `partial` полезен, когда фиксированные аргументы делают код понятнее.
- `reduce` не всегда улучшает читаемость: иногда обычный цикл, `sum`, `max`, `min` или `join` лучше.
- `singledispatch` удобен для простых случаев диспетчеризации по типу, но не должен заменять нормальную архитектуру.
